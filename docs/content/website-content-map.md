# Website Content Map (Revision v0.0.2)

## Purpose

This document tracks route coverage, longform page expectations, and content-linking requirements for the main web app.

Goals for this revision:
- Every primary section has a dedicated page.
- Internal links resolve to existing routes.
- Detail pages include recommended reading, further reading, and similar topics.
- Civilization coverage explicitly includes Sumerian, Babylonian, and Mayan entries.
- Library books include Amazon cover images and purchase links with affiliate tag support.

## Data Source

Core data is centralized in `apps/web/src/app/content.ts`.

Data groups:
- `BOOKS`
- `CIVILIZATIONS`
- `LEARNING_PATHS`
- `GLOSSARY_TERMS`

Shared helpers:
- `getAmazonLink(asin, associateId?)`
- `CIVILIZATION_SLUG_ALIASES`

## Book Coverage and Affiliate Links

Each book entry includes:
- canonical metadata (title, author, period, civilization)
- summary and long description
- themes
- Amazon ASIN
- Amazon cover URL
- related book slugs

Affiliate behavior:
- Purchase links use `https://www.amazon.com/dp/{ASIN}?tag={ASSOCIATE_ID}`
- `NEXT_PUBLIC_AMAZON_ASSOCIATE_ID` is supported when present
- default fallback tag is `livinglibrary-20`

## Detail Page Requirements

### Library detail pages
Must include:
- longform orientation text
- core themes
- recommended reading
- further reading
- similar topic links

### Civilization detail pages
Must include:
- context summary
- foundational text corpus
- core teachings
- recommended reading cards
- similar civilizations/topics

### Learning path detail pages
Must include:
- path metrics
- outcomes
- recommended reading
- study flow guidance
- similar paths/topics

### Glossary detail pages
Must include:
- definition and context
- related terms
- further reading links

## SEO Baseline

Detail pages use route-aware metadata generation (`generateMetadata`) with:
- unique page titles
- descriptive summaries
- intent-specific wording (overview, context, reading guidance)

## Content Operations Guidance

For future updates:
1. Add new civilization/book/path/term only in `content.ts` first.
2. Ensure new slugs are linkable from at least one hub page.
3. Add at least one recommended link and one related-topic link per new item.
4. Keep affiliate links generated by helper to maintain consistency.
5. Re-run route and type checks after each data update.
